## 2. 背景与目标

### 2.1 背景

FPS玩家核心音频矛盾：**枪声炸耳 vs 脚步微弱**。市场方案（FxSound、Equalizer APO、SoundLock、赛睿GG）要么通用性差、要么门槛高、要么功能单一。

用户最强烈呼声（B站696条弹幕+评论）：
- "我不敢把声音调大，怕枪声太大了，脚步声就很小"（67赞）
- "能不能更新一个棱镜2的三角洲EQ，找遍全网都没有"
- "照抄就行""直接抄答案"

**差异化机会**：fxsound_engine.exe v2.0 已就绪（虚拟设备音频管线+游戏预设+驱动管理），缺一个**零门槛GUI外壳**。

### 2.2 目标

将 CLI 包装为"选游戏→一键开"的 Windows 桌面应用，让FPS普通玩家**5秒内获得专业级游戏音频优化**。

### 2.3 成功指标

| 指标 | 目标值 |
|------|--------|
| 日常启动到生效 | <5秒 |
| 预设一键应用成功率 | >95% |
| 崩溃后音频恢复率 | 100% |

### 4.2 核心交互

**P0 - 预设卡片**：3列卡片（PUBG/三角洲/无畏契约），点击即切换，运行中卡片绿色边框+脉冲动画。

**P0 - 启用/停止**：大按钮控制引擎进程。启用前自动检测驱动，缺失则引导安装。

**P0 - 驱动管理**：启动时自动执行 `--check-driver`，未安装则顶部黄色提示条，点击安装（需UAC提权）。

**P1 - 效果滑块**：5个滑块（保真度/环绕/氛围/动态增强/低音），范围0-10，跟随预设默认值，拖动后重启引擎生效。

**P1 - 设备选择**：执行 `--list-devices` 渲染设备列表，启动时传 `--device-index`。

## 6. 技术架构

```
Flutter App (Windows)
  |-- UI Layer: 主界面 + 设置页
  |-- State Layer: Riverpod (EngineState/PresetState/DriverState)
  |-- Service Layer:
  |     |-- EngineService: Process.start/kill, stdout JSON解析
  |     |-- FacParser: .fac二进制文件解析
  |     └── DriverService: --check-driver/--install-driver
  +-- fxsound_engine.exe (子进程, WASAPI音频管线)
```

**进程通信协议**：
- 启动：`fxsound_engine.exe --preset <path> [--fidelity N] ...`
- stdout：每行一个JSON（device_saved/crash_recovery/shutdown）
- 停止：Process.kill()，引擎自行恢复音频设备

## 9. 非功能需求

### 9.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 音频延迟 | < 20ms | DfxDsp处理延迟+WASAPI环路延迟总和 |
| CPU占用 | < 3%（单核） | 游戏同时运行时的引擎CPU占用 |
| 内存占用 | < 50MB | fxsound_engine.exe常驻内存 |
| 启动时间 | < 3秒 | 从点击应用到听到增强后音频 |
| Flutter UI响应 | < 100ms | 滑块拖动到参数生效的UI反馈延迟 |

### 9.2 兼容性要求

| 平台 | 要求 |
|------|------|
| Windows版本 | Windows 10 1903+ / Windows 11 |
| 架构 | x64 only（FxSound源码限制） |
| 音频设备 | 支持WASAPI的任意播放设备 |
| 游戏兼容性 | 覆盖Top 10 FPS游戏（PUBG/CS2/Valorant/Apex/OW2/R6S/战地/COD/Tarkov/三角洲） |

### 9.3 稳定性与鲁棒性

- **崩溃恢复**：引擎崩溃/强制终止后，下次启动自动检测恢复文件，恢复原始默认播放设备
- **驱动状态检测**：启动前检查fxvad.sys驱动是否已安装，未安装提示用户（MVP不自动安装，需手动）
- **进程保活**：Flutter端监听引擎stdout心跳，5秒无输出判定为异常，自动重启引擎
- **静默失败**：分析/诊断模式异常时输出JSON错误，不弹窗不崩溃
- **防误操作**：虚拟设备正在作为默认播放设备时，引擎禁止自行退出（必须先恢复默认设备）

### 9.4 安全要求

- 驱动使用Root-enumerated虚拟设备（非真实硬件驱动），反作弊检测风险极低
- 不修改游戏进程内存，不注入DLL，纯音频系统层面处理
- 不收集用户音频内容，不上传任何数据
- 开源合规：FxSound采用AGPL-3.0，方案C进程隔离模型满足许可证要求

### 9.5 用户体验要求

- **零配置**：首次安装后首次启动自动引导驱动安装，游戏列表预置热门游戏
- **一键切换**：从选择游戏到音效生效 ≤ 2次点击
- **即时反馈**：任何操作（切换预设/调整滑块）必须在100ms内给出视觉反馈
- **后台静默**：最小化到托盘，游戏全屏时不打扰

## 10. 数据模型与状态机

### 10.1 预设数据结构（Dart）

```dart
class AudioPreset {
  final String id;                    // 唯一标识（如"valorant_footsteps"）
  final String name;                  // 显示名称（如"无畏契约·听风者"）
  final String gameId;                // 关联游戏ID
  final String description;           // 一句话描述
  final String facFilePath;           // .fac预设文件绝对路径
  final Map<String, double> parameters; // 效果参数映射（ fidelity/surround/ambience/dynamicBoost/bass : 0.0-1.0）
  final List<String> tags;            // 标签：["脚步声增强", "FPS", "竞技"]
  final bool isBuiltIn;               // 是否内置预设
  final DateTime createdAt;
}

class GameProfile {
  final String id;                    // 游戏唯一标识
  final String name;                  // 游戏名称
  final String executableName;        // 可执行文件名（用于进程检测）
  final String? coverImagePath;       // 封面图路径
  final List<String> presetIds;       // 关联的预设ID列表
  final bool isInstalled;             // 本地是否检测到安装
}
```

### 10.2 引擎状态机

```
[Idle] --启动引擎--> [Initializing]
[Initializing] --驱动检查通过--> [DriverReady]
[Initializing] --驱动未安装--> [DriverMissing]
[DriverReady] --保存原设备+设置虚拟设备为默认--> [Active]
[Active] --加载预设--> [Processing]
[Processing] --调整参数--> [Processing]（保持）
[Processing] --停止引擎--> [Restoring]
[Restoring] --恢复默认设备--> [Idle]
[Processing] --异常退出--> [CrashRecovery]
[CrashRecovery] --恢复设备--> [Idle]
```

### 10.3 进程通信协议（JSON Lines）

**Flutter → Engine（stdin）**：
```json
{"cmd":"load_preset","path":"C:/.../valorant.fac"}
{"cmd":"set_param","param":"fidelity","value":8.5}
{"cmd":"set_device","name":"耳机"}
{"cmd":"stop"}
```

**Engine → Flutter（stdout）**：
```json
{"type":"status","state":"active","latency_ms":12}
{"type":"error","code":"DRIVER_NOT_FOUND","message":"fxvad.sys未安装"}
{"type":"device_saved","original_device":"Realtek High Definition Audio"}
{"type":"shutdown","status":"ok","device_restored":true}
```

### 10.4 本地存储（shared_preferences）

| Key | 类型 | 说明 |
|-----|------|------|
| `last_game_id` | String | 上次选择的游戏 |
| `last_preset_id` | String | 上次使用的预设 |
| `custom_presets` | JSON String | 用户自定义预设列表 |
| `device_preference` | String | 用户偏好的播放设备名 |
| `first_run` | bool | 是否首次运行 |
| `agree_eula` | bool | 是否同意EULA |
