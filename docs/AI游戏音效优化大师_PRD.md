## 2. 背景与目标

### 2.1 背景

FPS端游玩家对"听声辨位"能力有极强依赖——脚步声方向判断、枪声远近识别、环境音干扰过滤，直接决定对战胜负。然而现有解决方案存在严重体验断点：

| 痛点 | 现状 | 用户损失 |
|------|------|---------|
| 系统音频无优化 | Windows默认音频管线不做游戏场景优化 | 脚步声被环境音淹没，听不清方向 |
| 游戏内均衡器粗糙 | 大多数FPS仅提供高/中/低三段EQ | 无法精准调频，拉高脚步频段连枪声一起炸 |
| 独立声卡贵且复杂 | 创新Sound Blaster单价500+，驱动配置繁琐 | 硬件门槛高，小白不会调 |
| 专业软件臃肿 | FxSound/Equalizer APO面向音乐场景，非游戏优化 | 参数预设不匹配，调参成本高 |
| 反作弊风险 | 部分音频驱动被反作弊标记 | 职业比赛/排位被封号 |

**核心洞察**：FPS玩家需要的不是"更好的音质"，而是"更清晰的战场信息"。音乐发烧友追求的声场宽广、低音浑厚，在FPS里恰恰是干扰源——他们需要的是**脚步声增强、枪声抑制、方位清晰**。

### 2.2 目标

| 维度 | 目标 |
|------|------|
| 用户价值 | FPS玩家一键获得"超级听力"——脚步更远更清晰、枪声不炸耳、方位更精准 |
| 产品定位 | 面向端游FPS玩家的AI音频增强工具，零配置、一键式、反作弊友好 |
| 差异化 | 游戏场景专属调音 vs 通用音频增强；AI预设推荐 vs 手动EQ调节 |
| 商业模式 | MVP阶段免费使用，优先用户获取（PC端不考虑营收） |

### 2.3 成功指标

| 指标 | 定义 | MVP目标 |
|------|------|---------|
| 激活率 | 安装后首次启动引擎成功 | ≥90% |
| 一键使用率 | 选择预设后无需手动调参直接开始游戏 | ≥80% |
| 留存率(D7) | 安装后第7天仍活跃使用 | ≥40% |
| NPS | 用户净推荐值 | ≥50 |
| 音频延迟 | 引擎处理延迟 | ≤10ms（玩家无感） |
| 故障率 | 引擎异常退出/音频中断 | ≤1% |

### 4.3 功能详情

#### 4.3.1 一键增强（P0）

**需求描述**：核心交互——一键启动/停止音频增强引擎，零配置体验。

**交互流程**：
1. 用户点击主界面"开启增强"按钮
2. 应用检查驱动状态（未安装→引导安装流程）
3. 启动fxsound_engine.exe子进程（传入当前预设路径+设备参数）
4. 读取stdout JSON流，更新UI状态
5. 引擎就绪后，主界面显示"增强已开启"+ 运行时长
6. 用户点击"关闭增强"→ 发送关闭信号 → 引擎恢复设备 → 退出

**引擎启动参数**：
```
fxsound_engine.exe --preset "presets/pubg.fac" --device "耳机 (ROG CETRA II)"
```

**状态机**：

```
[未启动] → 点击开启 → [检查驱动] → 驱动OK → [启动引擎] → 引擎就绪 → [增强中]
                                              ↓                    ↑
                                          驱动未安装 → [引导安装] → 安装成功
[增强中] → 点击关闭 → [发送关闭信号] → [等待引擎退出] → [已停止]
[增强中] → 引擎异常退出 → [自动恢复设备] → [错误提示] → [未启动]
```

**异常处理**：
| 异常场景 | 处理方式 |
|---------|---------|
| 驱动未安装 | 弹窗引导"一键安装驱动"（需管理员权限） |
| 引擎启动失败 | 显示错误JSON中的message，提供"重试"按钮 |
| 引擎运行中崩溃 | 自动恢复系统音频设备，弹窗通知"引擎异常退出"，提供"重新启动" |
| 音频设备变更 | 引擎自动处理（防抖1秒），UI显示device_change通知 |
| Process.kill()强杀 | 依赖.fxsound_recovery崩溃恢复文件，下次启动自动恢复 |

**埋点需求**：
- `engine_start`：引擎启动（含preset_name, device_name）
- `engine_stop`：引擎停止（含duration_seconds）
- `engine_crash`：引擎异常退出（含error_type）

#### 4.3.2 游戏预设系统（P0）

**需求描述**：提供FPS游戏专属音频预设，每个预设针对特定游戏的音频特征优化。

**内置预设（MVP）**：

| 预设名称 | 目标游戏 | 核心策略 | 关键参数 |
|---------|---------|---------|---------|
| PUBG竞技 | PUBG | 极限脚步+大地图方位 | Fidelity 115, Surround 105, Ambience 15, Dynamic Boost 65, Bass 25 |
| 无畏契约竞技 | Valorant | 中近距离脚步+枪声抑制 | Fidelity 90, Surround 80, Ambience 20, Dynamic Boost 60, Bass 15 |
| CS2经典 | CS2 | 紧凑地图精准定位 | Fidelity 100, Surround 90, Ambience 25, Dynamic Boost 55, Bass 20 |
| Apex战场 | Apex Legends | 远距离枪声+团队沟通 | Fidelity 95, Surround 95, Ambience 30, Dynamic Boost 60, Bass 30 |
| 通用FPS | 其他FPS | 均衡增强 | Fidelity 80, Surround 70, Ambience 25, Dynamic Boost 50, Bass 20 |

**预设效果说明示例（PUBG竞技）**：
> 🎯 脚步声增强：草地/木板/水泥地脚步声最大+30%清晰度
> 🔇 枪声抑制：3.5kHz刺耳频段-7dB，长时间对战不疲劳
> 🧭 方位增强：环绕声+5%，8km大地图方向辨识更准
> 💥 低频控制：轰炸区/载具低频-12%，不掩蔽中高频脚步

**预设数据结构（FAC文件）**：
- 文件位置：`presets/` 目录下，与exe同分发包
- 格式：FxSound原生.fac文本格式
- 每个预设包含：6个Main参数（0-127）+ 7个Integer开关 + 10段EQ

**交互流程**：
1. 用户在预设下拉框选择游戏
2. 应用读取对应.fac文件路径
3. 如果引擎运行中→发送参数更新（重启引擎加载新预设）
4. 如果引擎未运行→记录选择，下次启动时使用
5. 预设描述区更新为当前预设的效果说明

**埋点需求**：
- `preset_select`：选择预设（含preset_name）
- `preset_switch`：运行中切换预设（含from_preset, to_preset）

#### 4.3.3 实时调参面板（P1）

**需求描述**：提供5维效果滑条，允许玩家在预设基础上微调。

**5维参数**：

| 参数 | 中文名 | 范围 | 默认值 | 游戏场景含义 |
|------|--------|------|--------|------------|
| Fidelity | 保真度 | 0-10 | 9 | 值越高，音频细节越多，脚步/换弹等微声越清晰 |
| Surround | 环绕声 | 0-10 | 8 | 值越高，3D空间感越强，方位判断越准 |
| Ambience | 氛围感 | 0-10 | 2 | 值越高环境混响越强，FPS中建议低值（减少干扰） |
| Dynamic Boost | 动态增强 | 0-10 | 6 | 值越高，小音量被放大越多，微声脚步更突出 |
| Bass | 低音增强 | 0-10 | 2 | 值越高低频越强，FPS中建议低值（减少低频掩蔽） |

> **注意**：滑条范围0-10为UI显示值，映射到FAC文件0-127范围（乘以12.7取整）。

**交互流程**：
1. 选择预设后，5个滑条自动加载预设值
2. 用户拖动滑条，实时将新值通过CLI参数传递给引擎
3. 调参方式：**重启引擎**（MVP）→ 后续版本改为引擎运行时热更新
4. 提供"恢复预设默认值"按钮

**参数传递方式**：
```
fxsound_engine.exe --preset "pubg.fac" --fidelity 9 --surround 8 --device "耳机"
```
> CLI覆盖参数优先级高于FAC文件值

**埋点需求**：
- `param_adjust`：调整参数（含param_name, old_value, new_value）
- `param_reset`：恢复默认值

#### 4.3.4 游戏自动检测与预设切换（P1）

**需求描述**：检测前台游戏窗口，自动切换对应预设。

**检测机制**：
1. 每2秒轮询前台窗口进程名（Windows API: `GetForegroundWindow` + `GetWindowThreadProcessId`）
2. 匹配游戏进程名映射表
3. 匹配成功且当前预设不同→自动切换预设
4. 匹配失败（非游戏窗口）→保持当前预设不变

**游戏进程映射表（MVP）**：

| 游戏名称 | 进程名 | 预设 |
|---------|--------|------|
| PUBG | TSLGame.exe | PUBG竞技 |
| 无畏契约 | VALORANT-Win64-Shipping.exe | 无畏契约竞技 |
| CS2 | cs2.exe | CS2经典 |
| Apex Legends | r5apex.exe | Apex战场 |
| 三角洲行动 | DeltaForce.exe | 通用FPS |

**交互**：
- 自动切换时显示托盘通知："已切换至PUBG竞技预设"
- 设置中可关闭自动切换
- 可添加自定义游戏（进程名+预设绑定）

**埋点需求**：
- `game_detected`：检测到游戏（含game_name, process_name）
- `auto_preset_switch`：自动切换预设（含game_name, preset_name）

#### 4.3.5 虚拟驱动管理（P0）

**需求描述**：管理fxvad.sys虚拟音频驱动的安装/卸载/检测。

**驱动说明**：
- 驱动文件：fxvad.sys（微软WDM架构虚拟音频设备驱动）
- 硬件ID：Root\FXVAD（Root-enumerated设备，**反作弊风险极低**）
- 功能：创建虚拟播放设备"FxSound Audio Enhancer"作为音频中转站
- 证书：需使用EV代码签名证书签名驱动（MVP阶段可用测试签名模式）

**首次安装流程**：
1. 启动应用→检测驱动→未安装
2. 弹窗提示"首次使用需安装音频增强驱动"
3. 用户点击"一键安装"→触发UAC提权
4. 调用 `fxsound_engine.exe --install-driver`（管理员权限）
5. 等待安装结果JSON
6. 成功→自动继续启动引擎；失败→显示错误信息

**驱动状态检测**：
- 启动时自动调用 `fxsound_engine.exe --check-driver`
- 读取JSON结果：`{"driver_installed": true/false}`
- 未安装→引导安装；已安装→继续

**异常处理**：
| 异常场景 | 处理方式 |
|---------|---------|
| 安装失败（权限不足） | 提示需管理员权限，引导UAC提权 |
| 安装失败（签名问题） | 提示需开启测试签名模式，提供操作指引链接 |
| 卸载需求 | 设置中提供"卸载驱动"选项（需管理员权限） |
| 驱动丢失 | 检测到驱动不在→重新引导安装 |

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 音频处理延迟 | ≤10ms | DfxDsp处理延迟，WASAPI缓冲默认40ms |
| 内存占用 | ≤50MB | 引擎进程+Flutter应用合计 |
| CPU占用 | ≤3%（空闲时） | 无音频播放时 |
| CPU占用 | ≤8%（游戏时） | 满负载音频处理 |
| 启动时间 | ≤3s | 从点击开启到引擎就绪 |
| 预设切换 | ≤2s | 从选择预设到新预设生效 |
| 安装包体积 | ≤30MB | 含驱动文件+预设文件 |

### 5.2 安全需求

| 需求 | 说明 |
|------|------|
| 反作弊兼容 | 使用Root-enumerated虚拟设备（Root\FXVAD），不Hook游戏进程，不注入DLL，不修改游戏内存，反作弊系统无感知 |
| 驱动签名 | fxvad.sys需WHQL签名或EV代码签名；MVP阶段可使用测试签名模式 |
| 数据安全 | 不收集用户音频数据，不联网传输音频流 |
| 权限最小化 | 仅在驱动安装/卸载时请求管理员权限，日常使用无需提权 |

### 5.3 兼容性需求

| 维度 | 要求 |
|------|------|
| 操作系统 | Windows 10 1903+ / Windows 11 |
| 架构 | x64 only |
| 音频设备 | 所有WASAPI兼容播放设备（USB耳机、3.5mm耳机、蓝牙耳机、扬声器） |
| 游戏兼容 | 不与游戏反作弊冲突（EAC、BattlEye、Vanguard等） |
| 音频软件 | 与Discord/TT语音等VoIP软件共存 |
| 并发 | 不与FxSound官方客户端同时运行（检测并提示） |

### 5.4 可靠性需求

| 需求 | 说明 |
|------|------|
| 崩溃恢复 | 引擎异常退出后，通过.fxsound_recovery文件自动恢复系统默认音频设备 |
| 信号处理 | 捕获Ctrl+C、关闭窗口、注销、关机等信号，确保设备恢复 |
| 进程隔离 | 引擎作为独立进程运行，引擎崩溃不影响Flutter应用 |
| 状态持久化 | 用户选择的预设、调参值持久化到本地配置文件 |

### 8.2 音频管线

```
游戏音频输出 → [FxSound虚拟设备(fxvad.sys)]
                      │
                      ▼ WASAPI Loopback Capture
              [DfxDsp 音频处理引擎]
              ├─ Fidelity: 细节增强
              ├─ Surround: 3D声场扩展
              ├─ Ambience: 环境混响控制
              ├─ Dynamic Boost: 动态压缩
              ├─ Bass: 低频调节
              └─ 10-Band EQ: 频段精调
                      │
                      ▼ WASAPI Render
              [真实播放设备(耳机/扬声器)]
                      │
                      ▼
                  玩家听到优化后的音频
```

### 8.3 进程通信协议

Flutter与引擎通过stdout JSON流通信：

| 消息类型 | 方向 | 用途 |
|---------|------|------|
| engine_ready | Engine→Flutter | 引擎启动成功 |
| engine_error | Engine→Flutter | 启动/运行错误 |
| device_saved | Engine→Flutter | 原始设备已保存 |
| device_change | Engine→Flutter | 音频设备变更通知 |
| crash_recovery | Engine→Flutter | 崩溃恢复通知 |
| shutdown | Engine→Flutter | 引擎正常退出 |
| process_exit | OS→Flutter | 进程退出（异常/强杀） |

### 8.4 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 引擎形态 | 独立CLI进程 | 进程隔离→引擎崩溃不影响应用；AGPL合规（不链接） |
| 音频捕获 | WASAPI Loopback | 零反馈零杂音，系统级音频拦截 |
| 虚拟设备 | fxvad.sys (Root\FXVAD) | Root-enumerated，反作弊无感知 |
| 进程通信 | stdout JSON | 简单可靠，无需额外IPC框架 |
| 配置文件 | FAC文本格式 | FxSound原生格式，解析代码已可用 |
| 设备恢复 | .fxsound_recovery文件 | 兜底Process.kill()场景，不依赖注册表 |

## 9. 风险评估

| 风险类型 | 风险描述 | 影响程度 | 应对措施 |
|---------|---------|---------|---------|
| 技术风险 | fxvad.sys驱动在某些Windows版本安装失败 | 高 | 提供详细安装诊断；备选方案：改用软件虚拟设备（不用驱动） |
| 技术风险 | 音频延迟过高影响竞技体验 | 高 | 默认40ms缓冲区，可调至20ms；持续优化WASAPI缓冲策略 |
| 合规风险 | AGPL-3.0许可证传染性 | 中 | 进程隔离模型限制传染范围；分发时附源码或获取方式 |
| 合规风险 | 驱动签名问题（未WHQL认证） | 中 | MVP使用测试签名模式；正式版申请EV代码签名 |
| 业务风险 | 反作弊系统误判 | 高 | 使用Root设备不Hook游戏进程；主动与EAC/BattlEye沟通白名单 |
| 业务风险 | 与FxSound官方客户端冲突 | 低 | 启动时检测FxSound进程，提示关闭 |
| 体验风险 | 虚拟设备切换导致短暂无声 | 中 | 切换时显示进度提示；3秒内完成 |
| 体验风险 | 退出后系统音频未恢复 | 高 | 三重恢复机制：信号处理+崩溃恢复文件+手动恢复CLI |

## 11. 附录

### 11.1 术语表

| 术语 | 解释 |
|------|------|
| FAC | FxSound预设文件格式，文本格式存储音频效果参数 |
| fxvad.sys | FxSound虚拟音频设备驱动，创建虚拟播放设备 |
| WASAPI | Windows Audio Session API，Windows低延迟音频接口 |
| Loopback | WASAPI捕获模式，录制系统音频输出而非麦克风输入 |
| DfxDsp | FxSound数字信号处理核心，执行5维增强+EQ处理 |
| Root-enumerated | Windows设备枚举方式，由驱动自行声明硬件，不需要真实PCI/USB设备 |
| ITD | 双耳时间差，声音到达左右耳的时间差异，是方位判断的核心线索 |
| ILD | 双耳响度差，声音在左右耳的响度差异，配合ITD判断方位 |
| Fidelity | 保真度，音频细节保留/增强程度 |
| Surround | 环绕声，3D空间声场扩展程度 |
| Ambience | 氛围感，环境混响/空间反射效果 |
| Dynamic Boost | 动态增强，压缩动态范围使小音量更突出 |
| Bass | 低音增强，低频段增益 |
